# Data Flow Execution Steps

## 1. Offline State Execution

- The cashier completes a checkout inside the Flutter UI.
- The application checks network availability and detects an offline state.
- Transaction payloads are committed directly into the SQLite/ObjectBox local database.
- A background synchronization task is scheduled inside the application.

---

## 2. Reconnection & Transmission

- The internet connection returns to the device.
- The Sync Manager wakes up and reads pending logs from local storage.
- Transactions are transmitted sequentially to the API Gateway through HTTPS POST requests.

### Critical Control

Each transaction payload contains a unique UUID generated by the POS client to enforce idempotency.

This prevents:
- Double billing
- Duplicate transaction replay
- Partial sync corruption during unstable network conditions

---

## 3. Ingestion Layer Processing

- The POS Backend Service receives the request and validates the client signature.
- Raw transaction data is immediately persisted into the primary PostgreSQL ledger database.
- The service responds to the POS client with:
  - `200 OK`
  - or `201 Created`
- The POS client clears the successfully synchronized transaction from the local offline queue.

---

## 4. Downstream ERP Synchronization

- The POS Backend publishes a `POS_Sale_Synced` event into Apache Kafka.
- The ERP Core Service asynchronously subscribes to the Kafka topic.
- The ERP:
  - Parses the payload
  - Performs tax transformations
  - Posts accounting entries into the financial database
- The Inventory Service consumes the same Kafka event and:
  - Decrements stock quantities
  - Updates warehouse inventory balances in real time

# Key Architectural Benefits

| Feature | Benefit |
|---|---|
| Offline-first POS | Transactions continue during internet outages |
| Local Database Queue | Prevents transaction loss |
| Idempotent UUID Sync | Prevents duplicate billing |
| Kafka Event Streaming | Enables scalable asynchronous processing |
| Microservice Separation | Independent ERP and Inventory scaling |
| PostgreSQL Ledger | Reliable financial audit trail |
| API Gateway | Centralized security and rate limiting |

- UUID storage policy

- Retry-safe synchronization flow

- Partial warehouse synchronization
